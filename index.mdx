Didit ID lets you issue and verify verifiable credentials across tenants. Use it to create credential offers, issue SD-JWT VCs, request wallet presentations, and verify disclosed claims with cryptographic and trust checks.

Didit ID implements OpenID4VCI 1.0 for issuance and OpenID4VP 1.0 for verification with DCQL. Credentials use SD-JWT VC draft-16 with the `application/dc+sd-jwt` media type, holder key binding, and IETF Token Status List revocation.

## What you can build

- Issue SD-JWT VC credentials through OpenID4VCI credential offers.
- Verify wallet presentations through OpenID4VP and DCQL.
- Check signatures, key binding, audience, nonce, algorithm policy, expiry, revocation, and issuer trust.
- Manage schemas, templates, issued credentials, trusted issuers, relying parties, issuer keys, and status lists.

## API basics

The development base URL is `http://localhost:8011/v1`. Management endpoints use `Authorization: Bearer <tenant API key>`. OpenID4VCI and OpenID4VP protocol endpoints are public.

Start with [authentication](/reference/authentication), then follow [issuance](/guides/issuance) or [verification](/guides/verification).
